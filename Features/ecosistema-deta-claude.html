02 — Sistema de Skills

Instrucciones persistentes
por dominio

Un skill es un archivo que define cómo Claude Code debe abordar una categoría de tareas. Se cargan bajo demanda y acumulan contexto real del proyecto.

¿Cómo funciona? Cuando el usuario menciona una tarea que corresponde a un skill (ej. "crea un post para el blog"), Claude carga el skill deta-content que contiene la voz de marca, las fórmulas de copy, las keywords y el funnel. Sin el skill, respondería de forma genérica. Con él, responde como si conociera el proyecto de memoria.

03 — Casos de uso prácticos

Qué pasa exactamente
cuando pides algo

Ejemplos reales de cómo funciona el flujo — desde la instrucción hasta el entregable.

✍️

Post nuevo para el blog

Claude + deta-content + deta-blog-research

  • 1 El usuario pide un tema: "escribe sobre cómo delegar sin perder el control"
  • 2 Se carga deta-blog-research para analizar keywords con pytrends MX
  • 3 Se carga deta-content para aplicar la voz de marca y la fórmula de post
  • 4 Se genera el post con H1 orientado al problema del lector, estructura H2/H3, y CTA al Diagnóstico Express al final
  • 5 Se crea el archivo MDX en app/blog/posts/ con metadatos SEO completos
deta-blog-research deta-content coding
👤

Perfil de puesto para cliente

Claude Code o Cowork + perfiles-puesto

  • 1 El usuario da el nombre del puesto y contexto del cliente
  • 2 Se carga perfiles-puesto — define exactamente 5 áreas de efectividad y 5 competencias
  • 3 Claude genera el HTML con el template oficial y las secciones "Sí le corresponde / No le corresponde"
  • 4 Se convierte a PDF con node html_to_pdf.mjs (Puppeteer) — fidelidad visual garantizada
  • 5 El PDF queda listo para compartir con el cliente
perfiles-puesto Puppeteer Drive
🧑‍💼

Análisis de candidato

Cowork + entrevista + NotebookLM

  • 1 El CV y notas de entrevista se indexan en NotebookLM con el skill notebooklm
  • 2 Se activa el skill entrevista en Cowork — extrae competencias, evalúa fit, identifica riesgos
  • 3 Cowork genera el HTML del reporte usando deta_pdf_base.py (ReportLab)
  • 4 El PDF final se entrega al equipo DETA para revisión antes de enviar al cliente
entrevista notebooklm Cowork deta_pdf_base.py
🎨

Nueva sección o página web

Claude Code + coding + deta-brand

  • 1 El usuario describe la sección: "agrega un bloque de preguntas frecuentes en /reclutamiento"
  • 2 Se carga deta-brand para garantizar tokens correctos (colores, espaciado, tipografía)
  • 3 Se carga coding para respetar la arquitectura Next.js y las convenciones del proyecto
  • 4 El componente se crea con Tailwind, se prueba build limpio con tsc
  • 5 Push a main → Vercel hace el deploy automático
coding deta-brand Vercel
🔍

Auditoría de accesibilidad

Claude Code + a11y-audit + performance-audit

  • 1 El usuario pide: "revisa el sitio, quiero saber si hay problemas de contraste"
  • 2 Se activa deta-web-audit como orquestador
  • 3 a11y-audit analiza contrastes, etiquetas ARIA, semántica HTML — entrega un listado priorizado
  • 4 Claude aplica los fixes directamente en los archivos afectados
  • 5 Build limpio confirmado, commit con descripción clara, push
deta-web-audit a11y-audit performance-audit
📊

Reporte de pool de candidatos

Cowork + reporte-pool + deta_reporte_pool.py

  • 1 Se tienen 3-5 candidatos evaluados y el usuario pide el reporte comparativo
  • 2 Se carga reporte-pool — define estructura, matriz de comparación y criterios de presentación
  • 3 deta_reporte_pool.py genera el PDF con la comparativa, ranking y recomendación
  • 4 El reporte se entrega al cliente como PDF con diseño DETA — listo para presentar
reporte-pool deta_reporte_pool.py Cowork

06 — Principios de diseño

Cómo se ve todo lo que
produce DETA

Los 6 mandamientos que guían cualquier pieza visual — web, PDF, infografía o asset de redes.

01

1 idea por viewport

Si hay más de un pensamiento en pantalla, hay más de una sección. El scroll es narrativa, no lista de elementos.

02

Whitespace = confianza

El espacio vacío no es desperdicio — es el lenguaje del premium. 80px entre secciones en desktop, siempre.

03

La jerarquía se siente

Si hay que leer para entender qué es más importante, la jerarquía falló. Se detecta en 3 segundos.

04

El copy es diseño

Un headline débil arruina el mejor layout. El H1 describe el problema del cliente — nunca lo que hace DETA.

05

CTAs claros, no ansiosos

Máximo 1 CTA por sección. Invitación, no demanda. El texto describe exactamente qué pasa al hacer clic.

06

Reducir, no añadir

Cuando hay dudas, se quita. Si no comunica, no está. Cero decoración vacía, cero gradientes de 4 colores.
